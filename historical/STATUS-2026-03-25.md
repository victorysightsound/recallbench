## What's Built in Femind

### Ingest
- `store()` — single memory with optional embedding
- `store_batch()` — batch with batched embedding
- `store_with_extraction()` — LLM extracts individual facts, stores each, creates graph edges
- Chunking: configurable size (500/1000/2000) with overlap
- Fact extraction: regex (structured data) + LLM (conversational text)

### Search
- FTS5 OR-mode with stop-word removal
- Vector similarity (brute-force cosine, ANN built but not wired)
- Hybrid RRF fusion with configurable k-values (100/20 for questions)
- Multi-query: original + key-phrase variant
- Graph filtering: demote superseded results
- Recency weighting: configurable boost for newer content
- Session diversification: configurable max per session
- Context assembly within token budget

### Feature Toggles (EngineConfig)
- `embedding_enabled` — on/off (FTS5 still works when off)
- `graph_enabled` — on/off (master switch for graph create + query)
- `dedup_enabled` — on/off (content hash dedup)
- `vector_search_mode` — "exact" / "ann" / "off"

### Search Tuning (AssemblyConfig)
- `max_per_session` — 0 (unlimited) to N
- `recency_boost` — 0.0 to 1.0
- `search_limit` — max results
- `graph_depth` — 0 (off) to N hops

### Backends
- **Embedding:** CandleNativeBackend (local MiniLM), ApiBackend (DeepInfra), FallbackBackend
- **LLM:** ApiLlmCallback (OpenAI-compatible), CliLlmCallback (Claude/ChatGPT CLI)
- **ANN:** instant-distance HNSW (feature-gated)

## What's Built in RecallBench

### Test Harnesses
- **retrieval-test** — search quality measurement (Recall@K, MRR, Hit Rate), zero LLM cost
- **extraction-test** — LLM extraction quality measurement
- **pipeline-test** — full pipeline with every feature toggleable via CLI flags

### Infrastructure
- Session cache: raw dataset storage (LongMemEval, ConvoMem, MAB)
- Embedding cache: per dataset/variant/chunk_size/model
- Result saving: JSON with config snapshot for comparison

### Systems
- `femind` — local CandleNativeBackend
- `femind-api` — DeepInfra API embedding
- `femind-mab` — single_document config (no diversification, recency)
- `femind-extract` — full LLM extraction pipeline
- `femind-fallback` — API with local fallback
- `memloft` — memloft-core adapter (built, untested)
